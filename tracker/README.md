# Lumina Tracker CLI

A command-line tool for generating cryptographic proofs and submitting impact entries to the **LuminaLedger** smart contract.

## Prerequisites

- Node.js ≥ 18
- `npm install` in this directory

## Quick Start

```bash
# Install dependencies
npm install

# Show help
node src/index.js --help

# Generate a proof (no blockchain needed)
node src/index.js proof --category energy --quantity 50

# Submit in mock mode (dry-run)
node src/index.js submit --category energy --quantity 50 --mock

# Categorise a spend description (mock AI)
node src/index.js categorize --description "Installed solar panels on rooftop" --mock
```

## Commands

### `submit`

Generate a proof and submit it to the ledger.

```
Options:
  -c, --category <type>           energy | capital | behavior  (required)
  -q, --quantity <number>         Measured value (kWh, USD cents, units)  (required)
  -r, --rpc <url>                 JSON-RPC endpoint  [env: RPC_URL]
  -k, --key <privateKey>          Signer private key  [env: SIGNER_KEY]
  -a, --address <contractAddr>    LuminaLedger contract address  [env: LEDGER_ADDRESS]
      --mock                      Dry-run – skip blockchain transaction
```

**Example (live submission):**

```bash
export RPC_URL="https://sepolia.infura.io/v3/<KEY>"
export SIGNER_KEY="0xabc123..."
export LEDGER_ADDRESS="0xDeployedContractAddress"

node src/index.js submit -c energy -q 120
```

### `proof`

Generate a proof hash offline, without submitting.

```
Options:
  -c, --category <type>    energy | capital | behavior  (required)
  -q, --quantity <number>  Measured value  (required)
      --json               Output as JSON
```

### `categorize`

AI-assisted categorization of a spend description via the Groq API (falls back to rule-based mock if no API key is provided).

```
Options:
  -d, --description <text>   Natural-language description  (required)
      --api-key <key>        Groq API key  [env: GROQ_API_KEY]
      --mock                 Use local rule-based classifier
```

### `leaderboard`

Print the top users ranked by cumulative impact score.

```
Options:
  -r, --rpc <url>             JSON-RPC endpoint
  -a, --address <addr>        LuminaLedger contract address  (required)
  -n, --top <number>          Number of entries to show  [default: 10]
```

## Environment Variables

| Variable          | Description                          |
|-------------------|--------------------------------------|
| `RPC_URL`         | JSON-RPC provider endpoint           |
| `SIGNER_KEY`      | Private key for signing transactions |
| `LEDGER_ADDRESS`  | Deployed LuminaLedger address        |
| `GROQ_API_KEY`    | Groq API key for AI categorization   |

## Proof Generation Details

Proofs are generated by ABI-encoding `(category, quantity, timestamp, extra, salt)` and taking the keccak256 hash. This matches the on-chain verification logic.

For batch submissions a canonical Merkle tree is built from multiple leaves. The root is stored on-chain for privacy-preserving aggregation.
